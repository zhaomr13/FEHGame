# Godot Fighter Demo - Design Document

## Purpose

Create a minimal Godot project to demonstrate that textures generated by ssbp_viewer work correctly in Godot. This validates the SSBP → PNG frames → Godot atlas workflow.

## Core Concept

A bare-bones fighting game with two characters facing each other. The focus is on showcasing the converted animations, not gameplay depth.

## Features

### Characters
- **Player 1**: Diadora (left side)
- **Player 2**: ArmorAX (right side, auto-flipped to face left)

### Controls
| Input | Action |
|-------|--------|
| Left/Right Arrow | Move character |
| Space | Attack (plays Attack1 animation) |

### Animation States
- **Idle**: Plays when no input (loops)
- **Walk**: Plays when moving left/right
- **Attack**: Plays Attack1 once when Space pressed

### Visual Design
- Simple colored background
- Characters positioned facing each other
- No UI elements (health bars, etc.) - keep it minimal

## Technical Design

### Scene Structure
```
Main.tscn
├── Background (ColorRect)
├── Player1 (Character scene instance)
└── Player2 (Character scene instance)

Character.tscn
├── AnimatedSprite2D (uses Aseprite spritesheet)
└── Character.gd (state machine script)
```

### State Machine
```
Idle ──[move input]──> Walk ──[no input]──> Idle
 │                      │
 │                      │
 └──[attack input]──────┘──> Attack ──[anim done]──> Idle
```

### Files to Import
- `Diadora.png` + `Diadora.json` (from ssbp_VV/)
- `ArmorAX.png` + `ArmorAX.json` (from ssbp_VV/)

## Success Criteria
1. Godot project opens without errors
2. Both characters display with correct animations
3. Controls work (move + attack)
4. Animations play smoothly from the converted atlas

## Out of Scope
- Health/damage system
- Win/lose conditions
- Sound effects
- Particle effects
- Multiple levels
- Menu screens
